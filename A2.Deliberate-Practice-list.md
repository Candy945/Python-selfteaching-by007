## 15天自学入门

本节内容针对自学Python入门设计，基本采用李笑来《自学Python入门训练营》课程设计方案。Day0-Day14作业的间隔时间不宜超过3天，每次作业最合理的时间是24小时完成。

相信我，你并不孤独。

因为，李笑来《自学Python入门训练营》的300多位战友已经完成整个课程，并提交了[**课程作业**](https://github.com/selfteaching/selfteaching-python-camp/tree/master/exercises) 和学习总结（包括中途放弃的战友作业）。也就是说，当你遇到困难时，你有标准答案可以参考，你可以看看前辈的代码是怎么写的，也可以看看前辈们完成整个课程后的收获。





### 如何自学？

要阅读的材料很多，我来帮您划一下自学的重点：

* 在自学Python成长营里，想要打磨自学这门手艺，阅读官方文档就显得格外重要。这是按照李笑来《自学是门手艺》进行的自学设计。

* 在自学过程中，要反复对学习内容进行进行 **归纳整理**，每一天要对自已在学习过程遇到的 **问题和错误** 进行记录，并上传到作业文件夹。这是对学习的沉淀，是帮助自己强化记忆的有效方法。

* 当遇到各种问题时，你需要通过issue和Google搜索并解决问题，不断克服那些“过早引用”的知识点，整个过程会让你越来越习惯独立思考，系统打磨自学这门手艺。


### 带着问题前进，弄不懂的先跳过去

#### 英文资料看不懂，电脑环境搞不懂

1. 以完成作业为标准，一开始不要试图搞明白每个概念，否则会有很大的挫败感。做完还是懵比是正常情况，在之后的学习和作业里自然会理解。

2. 学编程的一个副作用就是学好英语，强烈建议只读英文文档。如果英文阅读确实很难，也可以采用中英文对照阅读，以提高阅读效率。

3. 如果遇到问题，第一步就是准确描述问题，第二步就是通过issue和Google搜索并解决问题。先在[李笑来「自学Python入门训练营」Issue](https://github.com/selfteaching/selfteaching-python-camp/issues) 中进行搜索。

#### 作业只能 Copy/参考 前辈的代码感觉很难过

1. 不是每个人一开始就能顺利编写出自己的代码，作为一名新人，不论你使用什么方式，只要你完成了今天的作业，并且你自己亲自确认提交了今天的作业，就是一个非常成功的进步

2. 学习一门知识或技能的最终目的是使用，而不是把每个细节都弄懂。绝大多数情况下，我们会被一个问题死死卡住，通过搜索、求助他人依然无解。遇到这种情况，不如先将问题放一放，然后继续学习其他内容，随着知识容量的扩大和知识点的贯通，之前一些无法解决的问题可能就解决了。

3. 编写代码实在感觉困难的同学，可以查看[**参考答案**](https://github.com/selfteaching/selfteaching-python-camp/tree/master/exercises)，这并不可耻，但一定要新手敲打一遍。想一下子掌握所有的知识点是不可能的，但是只要你动手做了，就一定能记住一些东西！

4. 建议大家给自己设置个时间点，如果超过多少时间做不出来，别憋着，就去参考下别人是怎么实现的，现在学习的重点是能捋清代码实现的逻辑。切勿死磕！以完成作业为目标！

**正确高效使用 Github 是学习编程的必备技能。**

Git是当今最好的版本控制软件，在软件工程中常被用来管理代码仓库（repository），对多人协作非常友好。

在Github上可以找到很多优秀的开源项目。Github是基于Git最好的在线协作平台，是所有程序员都会使用的代码托管平台，因此多数人选择在Github上公开自己的代码。比如，比特币源代码[https://github.com/bitcoin/bitcoin](https://github.com/bitcoin/bitcoin)，再比如,人工智能领域的人脸识别[https://github.com/ageitgey/face_recognition](https://github.com/ageitgey/face_recognition)。
