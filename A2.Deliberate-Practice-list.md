## 15天自学入门

本节内容针对自学Python入门设计，基本采用李笑来《自学Python入门训练营》课程设计方案。Day1-Day15作业的间隔时间不宜超过3天，每次作业最合理的时间是24小时完成。



### 如何自学？

要阅读的材料很多，我来帮您划一下自学的重点：

在自学Python成长营里，想要打磨自学这门手艺，阅读官方文档就显得格外重要。这是按照李笑来《自学是门手艺》进行的自学设计。

在整个自学过程中，你需要通过issue搜索并解决问题，不断克服那些“过早引用”的知识点，整个过程会让你越来越习惯独立思考，系统打磨自学这门手艺。

pencil2提问规则： 在学习过程中，遇到问题请先搜索[李笑来「自学Python入门训练营」Issue]（https://github.com/selfteaching/selfteaching-python-camp/issues）。若没有相同问题，请直接在咱们成长营[Issue](https://github.com/Candy945/Python-selfteaching-for-007/issues)内提交问题。


**特别强调：**完成了作业后，请及时提交成长日记，因为归纳总结整理是自学的关键。可以把自己在自学过程中的学习情况和感受记录下来，这是对学习的沉淀，是帮助自己强化记忆的过程。请相信，养成每日总结和梳理的好习惯，终身受益。

### 你会遇到哪些困难？

* 跟自己能力差距太大，英文文档看不懂，电脑环境搞的我快崩溃了

以完成作业为标准，一开始不要试图搞明白每个概念，否则会有很大的挫败感。做完还是懵比是正常的哈，在之后的学习和作业里就会慢慢地理解了~

学编程的一个副作用就是学好英语，会慢慢好起来的。何况作业里都提供了中文文档对照阅读，效率更高呀。

如果遇到搜索也解决不了的困难，及时反馈！切勿死磕！以完成作业为目标！

* 只能抄袭/参考前辈的代码感觉很难过，程序只能copy前辈的

1.不是每个人一开始就能顺利编写出自己的代码，作为一名新人，不论你使用什么方式，只要你完成了今天的作业，并且你自己亲自确认提交了今天的作业，就是一个非常成功的进步

2.可能大家在做完之后还是有很多不明白的地方，不急哈，有些东西属于提前引用，再往后学慢慢会理解的，建议大家在写完代码成功运行之后，可以尝试着给代码加标签，之后复习回看的时候也会有所帮助。

3.编写代码实在感觉困难的同学，可以试着在网上搜例子参考，这并不可耻，但一定要手动敲打一遍。想一下子掌握所有的知识结构是不可能的，但是只要你动手做了，就一定能记住一些东西！

4.建议大家给自己设置个时间点，如果超过多少时间做不出来，别憋着，就去参考下别人是怎么实现的，现在学习的重点是能捋清代码实现的逻辑。

## DAY1
### 使用Github进行协作

### 学习目的

**掌握Github的基本使用是学习编程的必备技能。**

Git是当今最好的版本控制软件，在软件工程中常被用来管理代码仓库（repository），对多人协作非常友好。

在Github上可以找到很多优秀的开源项目。Github是基于Git最好的在线协作平台，是所有程序员都会使用的代码托管平台，因此多数人选择在Github上公开自己的代码。比如，比特币源代码[https://github.com/bitcoin/bitcoin](https://github.com/bitcoin/bitcoin)，再比如,人工智能领域的人脸识别[https://github.com/ageitgey/face_recognition](https://github.com/ageitgey/face_recognition)。

所以，我们的作业提交主要在Github上进行，目的是帮助大家熟练使用Github。


### 任务清单

#### 1. 注册一个Github账户。

* 通过[Github注册链接https://github.com/](https://github.com/) 创建一个Github账户，将**用户名**发送在**作业交流群**[『不出局』自学Python成长营](https://h5.dingtalk.com/invite-page/index.html?bizSource=____source____&corpId=dingace87ab0796a5ca635c2f4657eb6378f&inviterUid=37409030D404C7EDFF4BBDCA828801C7&encodeDeptId=0054DC2B53AFE745)，并@Candy945

#### 2. 创建一个自己的代码仓库（Create a new repository），学会使用分支（Branch)和 Pull Request。

* 阅读 [参考资料1](#参考资料)
* 创建一个名为 Growth-Diary 的仓库，记得勾选“Public" 和 “Initialize this repository with a README”，完成后在仓库内再创建一个 <007> 分支。
* 在 <007> 分支中创建一个名为 <成长日记.md> 的文件，提交一个 commit
* 从 <007> 分支提交一个 Pull Request 到 master 分支，并完成合并（merge）
* 在 master 分支中再创建一个名为 <学习资料.md> 的文件，提交一个 commit
* 从 master 分支提交一个 Pull Request 到 <007> 分支，并完成合并（merge）

#### 3. 在Github的仓库（repository）中提交 Issue。

* 阅读 [参考资料2](#参考资料)
* 在自己创建的 Growth-Diary 仓库中提交一个 Issue
* 为自己创建的 Issue 打上【help wanted】 标签


#### 4. 在自己的电脑上安装 Github Desktop 和 VS Code 客户端，用于管理仓库（repository）。

* 阅读 [参考资料3,4](#参考资料)
* 通过 官方下载地址[https://desktop.github.com/](https://desktop.github.com/) 安装 Github Desktop 桌面客户端，完成后 Log In 登陆自己的Github账户
* 下载安装 Anaconda 【Python 3.7 version】 [下载地址](https://www.anaconda.com/distribution/)（Anaconda 已包含 Python 运行环境，无需额外安装 Python）
   * 打开 Anaconda 的可视化操作界面，找到并安装 VS Code ，Jupyter Notebook 和 JupyterLab
   * 阅读 [VSCode官方教程](https://code.visualstudio.com/docs/python/environments) ，并在 VS Code 中完成 Python 环境配置

* 通过 Github Desktop 将 Growth-Diary 仓库 Clone 到本地电脑
* 通过 VS Code ，在本地电脑的 Growth-Diary 仓库中创建一个名为 <任务清单.md> 的文件
* 通过  Github Desktop 桌面客户端 ，将本地仓库新增的 <任务清单.md> 的文件提交为一个 commit
* 通过  Github Desktop 桌面客户端 ，将本地仓库的变更推送（Push）
