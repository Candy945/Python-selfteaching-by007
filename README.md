#只能抄袭/参考前辈的代码感觉很难过，程序只能copy前辈的

1.不是每个人一开始就能顺利编写出自己的代码，作为一名新人，不论你使用什么方式，只要你完成了今天的作业，并且你自己亲自确认提交了今天的作业，就是一个非常成功的进步

2.可能大家在做完之后还是有很多不明白的地方，不急哈，有些东西属于提前引用，再往后学慢慢会理解了，建议大家在写完代码成功运行之后，可以尝试着给代码加标签，之后复习回看的时候也会有所帮助。

3.编写代码实在感觉困难的同学，可以试着在网上搜例子参考，这并不可耻，但一定要手动敲打一遍。一下子能掌握所有的知识结构是不可能的，但是只要你动手做了，就一定能记住一些东西！

4.建议大家给自己设置个时间点，如果超过多少时间做不出来，别憋着，就去参考下写别人是怎么实现的，现在学习的重点是能捋清代码实现的逻辑~

#觉得打卡好麻烦

打卡的目的是为了让大家养成随时记录问题和总结每日所学知识的习惯，所以即使作业没有完成，也建议大家进行打卡总结。这是对学习的沉淀，是帮助自己强化记忆的过程！所以请一定在自学中养成这个习惯！

教练主要是负责课程的研发以及学员在学习中遇到问题去issue上面提问，助教负责学员在作业课程过程中自己无法独立解决的情况下，帮助学员学习，辅导员主要负责日常事务的沟通交流，解决一些疑惑，包括打卡以及发作业，督促和跟近学员学习状况

学员须知
真实的世界没有老师，人人都应自学，才能够突破自己。

自学很简单，其实就是不断习得、积累、研究、打磨、升级那些概念和方法论。进一步来看，自学能力是靠积累增强的，这个训练营就是加速积累的过程。

很多人错误地认为：所谓“自学”，就是“自己一个人（默默地）学”——这恰恰是绝大多数人一生学习失败的最根本原因。学习也好，进步也罢，从来都不是单独鼓励的行为，而是社交行为。

学习群中大家相互学习、互相鼓励，遇到问题积极提问，同时帮助他人解答问题，降低学习的孤独感和受挫感，体验高效社交。

为保证学习效果，请尽量抽出更多的时间学习，建议合理安排时间段，当天能花2-5个小时全身心投入。毕竟在自学营内，有辅导员和教练在，你学的越多就越是赚。

但注意，千万不要当伸手党。教练也好，社群内的其他同学也好，在这个自学训练营中，都是陪伴你自学的角色。一定要改变被动接受知识的状态，主动的阅读官方文档习得Python知识。教练不能手把手的教你Python是什么，督促你通过自学和社交解决问题，这才是训练营的主旨。
